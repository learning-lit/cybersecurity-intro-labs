Defense Briefing

System Hardening involves securing an operating system's configuration to minimize its attack surface. Standard out-of-the-box configurations are notorious for using insecure ports, enabling dangerous admin privileges, and keeping vulnerable legacy protocols active.

WORKSPACE: SSH Hardening Editor

Configuration File: /etc/ssh/sshd_config

Review our host's SSH server parameters. Attackers can brute-force root access if configurations permit root logins, default ports, and insecure password authentication.

Configuration Directives
Port Change the default SSH port (22) to block automated network sweeps.
PermitRootLogin Never allow root user access directly over SSH interfaces.
PasswordAuthentication Mandate cryptographic keys instead of vulnerable user passwords.
X11Forwarding Disable graphical interface forwarding to save network overhead and limit attack vectors.
